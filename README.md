# Экспертная система диагностики сетевых неполадок (на базе CLIPS)

Прототип интерактивной экспертной системы, предназначенный для автоматизации первой линии технической поддержки (Helpdesk) ИТ-инфраструктуры предприятия. Разработан в рамках прохождения производственной практики в качестве демонстрации применения декларативного программирования и продукционных моделей знаний.

## Описание проекта
Данная система аккумулирует знания сетевых инженеров и переводит сложную техническую диагностику (анализ IP-адресов, шлюзов и DNS) на понятный для конечного пользователя язык визуальных признаков. Система опрашивает пользователя о подключения сетевого провода к ПК и роутеру, IP-адрес, присвоенный компьютеру и доступность сайтов. После каждого вопроса дается рекомендация по дальнейшим действиям.

## 🛠 Технологический стек
* **Язык БЗ:** CLIPS (C Language Integrated Production System)
* **Среда разработки:** CLIPS IDE v6.4+ / CLI-оболочка CLIPS

## Инструкция по установке и запуску

### Предварительные требования
У вас должна быть установлена среда **CLIPS IDE** или настроена утилита командной строки CLIPS. [Скачать CLIPS](https://sourceforge.net/projects/clipsrules/files/latest/download).

### Пошаговый запуск в CLIPS IDE
1. Склонируйте репозиторий или скачайте файл базы знаний `network.clp`.
2. Откройте **CLIPS IDE**.
3. Полностью очистите оперативную память среды перед стартом:
   ```clips
   (clear)
   ```
4. Загрузите файл базы знаний (замените путь на ваш актуальный, используйте прямые слэши `/`):
   ```clips
   (load "C:/your_path/network.clp")
   ```
   *Убедитесь, что среда вывела в консоль список правил и финальное сообщение `TRUE`.*
5. Инициализируйте начальное состояние системы:
   ```clips
   (reset)
   ```
6. Запустите машину логического вывода:
   ```clips
   (run)
   ```
